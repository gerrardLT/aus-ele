# 通用模式

## 骨架项目

实现新功能时:
1. 搜索经过实战检验的骨架项目
2. 使用并行代理评估选项:
   - 安全评估
   - 可扩展性分析
   - 相关性评分
   - 实现规划
3. 克隆最佳匹配作为基础
4. 在经过验证的结构内迭代

## 设计模式

### Repository 模式

在一致的接口后封装数据访问:
- 定义标准操作: findAll, findById, create, update, delete
- 具体实现处理存储细节(数据库、API、文件等)
- 业务逻辑依赖于抽象接口,而不是存储机制
- 支持轻松切换数据源,并通过模拟简化测试

### API 响应格式

为所有 API 响应使用一致的封装:
- 包含成功/状态指示器
- 包含数据负载(错误时可为空)
- 包含错误消息字段(成功时可为空)
- 为分页响应包含元数据(总数、页码、限制)

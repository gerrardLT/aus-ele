# 澳洲电网（AEMO NEM）历史数据爬取方案与注意事项

关于能否爬取澳洲国家电力市场（NEM）的历史交易数据，答案是**肯定可以**的。AEMO（Australian Energy Market Operator）实际上提供了相对开放的数据访问渠道，但直接通过脚本请求其主网页（如 `aemo.com.au`）可能会遇到反爬虫拦截（返回 403 Forbidden 错误）。

以下是关于如何爬取该数据以及需要注意的核心事项：

## 一、 数据获取的主要途径

1. **官方开放数据门户 (NEMWEB)**
   - AEMO 提供了一个专门用于数据分发的公共服务器：[NEMWEB](http://NEMWEB.com.au/) (FTP/HTTP 索引目录)。
   - 大多数历史交易数据、电价、需求量、调度数据都以 `CSV` 或包含 CSV 的 `.zip` 文件存放在该服务器中。
   - 这是官方推荐且无需特殊认证即可进行批量下载和脚本抓取的途径。

2. **成熟的开源 Python 库 (推荐)**
   - **NEMOSIS (NEM Open Source Information Service)**: 这是由新南威尔士大学（UNSW）开发的一个生产级 Python 库，专门用于下载和处理 AEMO 发布的大量历史数据。它可以处理电价、FCAS、区域需求和发电机调度数据等，强烈推荐直接使用该库进行历史数据的拉取。
   - **nemdata**: 另一个针对研究人员的实用工具，用于从 MMSDM 数据库下载各类 NEM 数据集。
   - **nem-reader**: 用于解析 AEMO 特定的 NEM12（区间计量数据）和 NEM13 格式的数据文件。

3. **AEMO 官方 RESTful APIs**
   - AEMO 为市场参与者提供了一套官方的 APIs。但是，使用这些 API 通常需要**企业认证和市场参与者资质**。如果你仅仅是做数据分析或学术研究，优先依赖 NEMWEB 和上述开源库。

4. **主站网页直接爬取**
   - 主站网页（即你提供的链接）上可能内嵌了一些动态图表（通过 `<script>` 或 API 调用加载）。如果必须要抓取这些特定图表背后的实时 JSON 数据，可以通过浏览器开发者工具抓包，或者使用 Selenium/Playwright 模拟浏览器访问来绕过主站的 403 防护，但这不适合大规模历史数据的获取。

## 二、 爬取时需要注意的事项（避坑指南）

### 1. 强力的反爬与防火墙过滤 (403 Forbidden)
正如尝试直接抓取 AEMO 主站时的结果，其主域名（`www.aemo.com.au`）部署了强力的 Web 应用防火墙（如 Cloudflare 或类似产品）。
- **对策**: 避免直接爬取 `aemo.com.au` 上的 HTML 页面。将脚本的目标指向 [NEMWEB 门户](http://nemweb.com.au/) 或使用已封装好请求伪装机制的开源库（如 NEMOSIS）。

### 2. 数据量极其庞大
NEM 数据精确到 **5分钟** 甚至 **秒级**（如 FCAS 数据）。日积月累，历史数据的文件体积非常惊人。
- **对策**: 
  - 做好服务器存储空间的规划。
  - 在下载和解析时，建议按月/按日分块拉取，边拉取边入库（如导入 PostgreSQL 或 ClickHouse），避免试图将几十 GB 甚至 TB 级别的 CSV 文件一次性加载到内存中。

### 3. 数据表结构与关联复杂度 (MMSDM)
AEMO 提供的数据基于 MMSDM (Market Management System Data Model)。这种关系型数据库表的结构非常复杂，含有上百张表。
- **对策**: 在着手写爬虫前，必须先查阅 AEMO 提供的 [MMSDM Data Model 文档](https://aemo.com.au/energy-systems/electricity/national-electricity-market-nem/data-nem/market-management-system-mms-data)，搞清楚你需要的数据（如是 `DISPATCHPRICE`, `TRADINGPRICE` 还是 `DISPATCHREGIONSUM`）分别在哪些具体的表和字段里。

### 4. 频率控制与礼貌爬取
虽然 NEMWEB 相对开放，但过度频繁的并发请求随时可能导致你的 IP 被封禁。
- **对策**: 
  - 在请求之间加入适当的延时（如 `time.sleep(1)`）。
  - 使用缓存机制，不要重复下载已经拉取过且未更新的历史 `.zip` 压缩包。

### 5. 时区问题 (Timezone)
- **对策**: NEM 市场的时间戳默认统一使用 **澳大利亚东部标准时间 (AEST, UTC+10)**，并且**不遵守夏令时 (No Daylight Saving Time)**。在进行时间序列数据对齐分析时，务必在代码中强制指定并处理好这一时区，否则数据会出现严重的错位。

## 三、 推荐的实施方案总结
如果您准备使用 Python 开展此项目，**强烈建议不要从零手写爬虫，也不要直接去强行突破主站网页的反爬验证**。

**最优路径是：**
1. 评估 `NEMOSIS` 库是否能直接满足您的历史数据需求（大概率可以，支持通过简单的 Python 脚本拉取和合并数据）。
2. 如果需要更底层的定制，直接编写 Python 脚本请求 `http://nemweb.com.au/Reports/Current/` 或 `/Reports/Archive/` 获取历史压缩包，利用 `pandas` 进行解压和数据清洗。

## 四、 法律与合规性说明（版权与使用条款）

通常情况下，爬取和使用 AEMO 公开的 NEM 数据**不会受到严厉的法律限制或起诉**，由于其作为公共基础设施监督者的性质，他们本身也服务于全社会的能源研究，但在合规性上需严格遵守以下约束：

1. **强制版权与归属声明 (Attribution)**
   - AEMO 明确规定，允许个人和企业出于任何目的（包括商业和非商业用途）使用其公开的资料和数据。
   - **核心法律要求**：您必须在您的软件代码、分析报告或数据图表的显著位置，明确标注该数据来源于 "AEMO (Australian Energy Market Operator)"。如果不进行归属声明而直接利用，属于违反其数据使用协议。

2. **区分“公开数据”与“企业内部数据”**
   - 必须确保您抓取的目标是向全世界公开访问的端口（例如 [NEMWEB](http://NEMWEB.com.au/) 前面提到的开放目录或不设认证的 API）。
   - 切勿试图通过漏洞提权、撞库或其他手段侵入那些需要“注册市场参与者资质（Participant Credentials）”的接口。访问了不该访问的数据，才可能构成严重的违法爬虫行为。

3. **系统破坏与恶意调用 (DDoS 风险)**
   - 尽管下载数据是允许的，但如果在爬虫脚本中毫无节制地增加并发请求（比如一秒数百次调用），导致 AEMO 服务器超载甚至宕机，这会被系统定性为“恶意攻击”并可能引发严重的司法跟进（依据各个国家的网络安全法）。务必在脚本的每次请求中按规定增加延迟休眠（例如 `delay > 1s`）。

4. **免责风险 (Disclaimer of Liability)**
   - 如果您出于商业化做量化交易，需注意：AEMO 对于提供数据的准确性、完整性明确列出了免责条款。若因其发布的数据包含延迟或错误，导致您的自动交易策略出现亏损，AEMO 会通过其免责声明拒绝承担任何责任。

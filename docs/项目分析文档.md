## 1. 功能总览

本系统是一个面向 **澳大利亚国家电力市场 (NEM)** 和 **西澳电力市场 (WEM)** 的全栈数据平台，涵盖数据采集、存储、分析和可视化四大核心环节。

### 1.1 数据采集层

| 爬虫模块 | 数据源 | 采集内容 |
|---------|--------|---------|
| `aemo_nem_scraper.py` | AEMO NEMWEB (MMS) | NEM 5分钟交易电价 + 8种 FCAS 调频服务价格 |
| `aemo_wem_scraper.py` | AEMO WEM Data Portal | WEM 参考交易电价 (Reference Trading Price) |
| `aemo_wem_ess_scraper.py` | AEMO WEM Data Portal | WEM ESS 辅助服务价格 (能量/调频/应急/RoCoF) |

### 1.2 分析可视化层

| 模块 | 功能 |
|------|------|
| **电价趋势** | 全年电价时序图 + 峰值包络采样 + 统计摘要 |
| **峰谷价差分析** | 滑动窗口 Peak/Trough 分析 (1h/2h/4h/6h)，计算 Gross/Net Spread |
| **BESS P&L 模拟器** | 瀑布图展示毛利差 → 扣除 RTE/Aux/网费/MLF/AEMO费/衰退 → 净利润 |

| **充电窗口雷达** | 24小时径向热力图: 最优充/放电时段、负电价概率 |
| **投资分析模型** | NPV / IRR / 回收期 / 逐年现金流 / 收入构成条形图 |

### 1.3 后台管理

- 通过 API 手动触发数据同步

## 6. 图表计算公式大全

### 6.1 电价趋势图 — 峰值包络采样 (Peak-Preserving Envelope Sampling)

**所在文件**: `server.py` → `get_price_trend()`

**目的**: 将数万条 5 分钟数据降采样至 ≤1500 个点，同时保留极端峰值和谷值

```
输入: N 条原始数据点, 目标返回 limit 个点
sample_step = N ÷ limit

将数据分为 bucket_id = (ROW_NUMBER - 1) ÷ sample_step

每个 bucket 输出两个点:
  点1: (bucket_start_time, MAX(price))  — 峰值
  点2: (bucket_start_time, MIN(price))  — 谷值 (仅当 max ≠ min 时)
```

**统计指标**（单次 SQL 扫描完成）:

```
neg_ratio     = (负价格记录数 ÷ 总记录数) × 100%
neg_avg       = AVG(price) WHERE price < 0
pos_avg       = AVG(price) WHERE price > 0
days_below_100 = COUNT(DISTINCT day) WHERE price < -100
days_above_300 = COUNT(DISTINCT day) WHERE price > 300
```

### 6.2 峰谷价差分析 — 滑动窗口算法

**所在文件**: `server.py` → `get_peak_analysis()`

**目的**: 对每天的 5 分钟数据，用滑动窗口找出最高/最低连续均价

#### 6.2.1 窗口大小

```
NEM (5分钟间隔, 12个/小时):
  1h = 12 intervals
  2h = 24 intervals
  4h = 48 intervals
  6h = 72 intervals
```

#### 6.2.2 核心算法 — O(n) 滑动窗口

```python
# 对每天的价格序列 prices[], 窗口大小 w_size:

window_sum = SUM(prices[0 : w_size])
best_max = window_sum
best_min = window_sum

for i in range(1, n - w_size + 1):
    window_sum += prices[i + w_size - 1] - prices[i - 1]   # O(1) 滑动
    best_max = MAX(best_max, window_sum)
    best_min = MIN(best_min, window_sum)

peak_{label}   = best_max ÷ w_size    # 最高连续均价
trough_{label} = best_min ÷ w_size    # 最低连续均价
```

#### 6.2.3 价差计算

```
Gross Spread (毛价差):
  spread_{Xh} = peak_{Xh} − trough_{Xh}

Net Spread (净价差, 扣除双向网费):
  net_spread_{Xh} = spread_{Xh} − 2 × network_fee
```

#### 6.2.4 聚合统计

```
avg_spread_{Xh}     = mean(所有交易日的 spread_{Xh})
avg_net_spread_{Xh}  = mean(所有交易日的 net_spread_{Xh})
max_spread_{Xh}      = max(所有交易日的 spread_{Xh})
min_spread_{Xh}      = min(所有交易日的 spread_{Xh})
```

### 6.3 BESS P&L 模拟器 — 利润瀑布图

**所在文件**: `BessSimulator.jsx`

**目的**: 展示从毛价差到净利润的逐级扣减过程

#### 6.3.1 瀑布计算步骤

```
输入参数:
  grossSpread     = 4h 年均毛价差 ($/MWh, 从 peak-analysis API 获取)
  rte             = 往返效率 (默认 85%)
  auxPct          = 辅助用电比例 (默认 2%)
  mlf             = 边际损耗因子 (默认 0.95)
  networkFee      = 单向网费 (默认 40 $/MWh)
  aemoFee         = AEMO 参与者费用 (默认 0.5 $/MWh)
  degradationCost = 衰退成本 (默认 5 $/MWh)

扣减步骤:
  ① RTE 损失      = grossSpread × (100 - rte) / 100
  ② 辅助用电损失  = grossSpread × auxPct / 100
  ③ 网费 (双向)   = networkFee × 2
  ④ MLF 损失      = grossSpread × (1 - mlf)
  ⑤ AEMO 费 (双向) = aemoFee × 2
  ⑥ 衰退成本      = degradationCost

  netProfit = grossSpread − ① − ② − ③ − ④ − ⑤ − ⑥
```

#### 6.3.2 年化收入估算

```
dailyRevenue  = netProfit × capacityMW × durationH × cyclesPerDay
annualRevenue = dailyRevenue × 365
```

### 6.4 循环成本分析 — 价差分布直方图

**所在文件**: `CycleCost.jsx`

**目的**: 判断每次充放电循环是否值得（价差 vs 电池衰退成本）

#### 6.4.1 直方图分桶

```
bucketSize = 25 ($/MWh)
bucket = FLOOR(daily_spread_4h ÷ 25) × 25

例: 价差 $78 → 归入 $75 桶
```

#### 6.4.2 盈利判定

```
profitableDays   = COUNT(日) WHERE spread_4h > degradationCost
unprofitableDays = totalDays − profitableDays
profitableRatio  = profitableDays ÷ totalDays × 100%

颜色逻辑:
  绿色 (可循环):  rangeNum ≥ degradationCost
  黄色 (边际):    rangeNum ≥ degradationCost × 0.7
  红色 (不值得):  rangeNum < degradationCost × 0.7
```

### 6.7 充电窗口雷达 — 24小时径向热力图

**所在文件**: `ChargingWindow.jsx` + `server.py` → `get_hourly_price_profile()`

#### 6.7.1 每小时统计 (SQL)

```sql
SELECT
    CAST(substr(settlement_date, 12, 2) AS INTEGER) as hour,
    ROUND(AVG(rrp_aud_mwh), 2) as avg_price,
    ROUND(MIN(rrp_aud_mwh), 2) as min_price,
    ROUND(MAX(rrp_aud_mwh), 2) as max_price,
    COUNT(*) as count,
    SUM(CASE WHEN rrp_aud_mwh < 0 THEN 1 ELSE 0 END) as neg_count,
    ROUND(AVG(CASE WHEN rrp_aud_mwh < 0 THEN rrp_aud_mwh ELSE NULL END), 2) as neg_avg
FROM trading_price_{year}
WHERE region_id = ?
GROUP BY hour
```

#### 6.7.2 负电价比例

```
neg_pct = (neg_count ÷ total_count) × 100%
```

#### 6.7.3 最优窗口识别

```python
# 按均价排序
sorted_hours = sort(hourly_data, by=avg_price, ascending)

bestCharge    = sorted_hours[0:4]   # 均价最低的4个小时 → 最佳充电窗口
bestDischarge = sorted_hours[-4:]   # 均价最高的4个小时 → 最佳放电窗口
```

#### 6.7.4 热力图颜色插值

```javascript
function interpolateColor(price, minPrice, maxPrice):
  if price ≤ 0:
    t = |price| / |minPrice|
    → 绿色系 rgb(30-t×20, 160+t×95, 80-t×30)  // 负价=可充电

  if price > 0:
    t = price / maxPrice
    if t < 0.5:  → 暖黄色 rgb(200+p×55, 180-p×80, 50-p×30)
    if t ≥ 0.5:  → 深红色 rgb(230+p×25, 80-p×50, 30-p×20)  // 高价=可放电
```

### 6.8 套利回测引擎

**所在文件**: `bess_backtest.py` → `backtest_arbitrage()` + `server.py` → `investment_analysis()`

#### 6.8.1 每日最优充放电 (滑动窗口法)

```
输入:
  prices[]        = 当日所有 5 分钟价格 (约 288 个)
  charge_intervals = duration_hours × 12   (如 4h → 48 个)
  capacity_mwh    = power_mw × duration_hours
  RTE             = 0.87 (往返效率)

步骤 1: 找最优充电窗口 (均价最低)
  for i in range(n - charge_intervals + 1):
    avg = mean(prices[i : i + charge_intervals])
    if avg < best_charge_avg:
      best_charge_avg = avg
      best_charge_start = i

步骤 2: 找最优放电窗口 (均价最高, 不与充电重叠)
  for i in range(n - charge_intervals + 1):
    if overlap(i, charge_window): continue
    avg = mean(prices[i : i + charge_intervals])
    if avg > best_discharge_avg:
      best_discharge_avg = avg

步骤 3: 计算收入
  charge_cost      = best_charge_avg × capacity_mwh
  discharge_income = best_discharge_avg × capacity_mwh × RTE
  net_revenue      = discharge_income − charge_cost

  仅当 net_revenue > 0 时计入年度收入
```

#### 6.8.2 年化汇总

```
annual_revenue    = Σ(所有正收益日的 net_revenue)
revenue_per_mw    = annual_revenue ÷ power_mw
avg_daily_revenue = annual_revenue ÷ trading_days
```

### 6.9 投资分析模型 (NPV / IRR / 回收期)

**所在文件**: `server.py` → `investment_analysis()` + `InvestmentAnalysis.jsx`

#### 6.9.1 输入参数

| 参数组 | 参数名 | 默认值 | 说明 |
|--------|--------|--------|------|
| **储能规格** | `power_mw` | 100 | 额定功率 (MW) |
| | `duration_hours` | 4 | 储能时长 (h) |
| | `round_trip_efficiency` | 0.87 | 往返效率 (87%) |
| | `degradation_rate` | 0.025 | 年衰减率 (2.5%) |
| **成本参数** | `capex_per_kwh` | 350 | CAPEX 单价 ($/kWh) |
| | `fixed_om_per_mw_year` | 12,000 | 固定运维 ($/MW/年) |
| | `variable_om_per_mwh` | 2.5 | 可变运维 ($/MWh) |
| | `grid_connection_cost` | 5,000,000 | 并网费用 (一次性) |
| | `land_lease_per_year` | 200,000 | 土地租赁 ($/年) |
| **财务参数** | `discount_rate` | 0.08 | 贴现率 (8%) |
| | `project_life_years` | 20 | 项目寿命 (年) |
| | `revenue_capture_rate` | 0.65 | 收入捕获率 (完美预见的65%) |
| | `fcas_revenue_per_mw_year` | 15,000 | FCAS 年收入 ($/MW) |
| | `capacity_payment_per_mw_year` | 0 | 容量补贴 ($/MW) |

#### 6.9.2 Step 1: 历史回测基准收入

```
对每个回测年份:
  gross_arbitrage = Σ(每日最优套利净收入)    // 使用 6.8.1 的算法

baseline_arbitrage = mean(各年 gross_arbitrage) × revenue_capture_rate
baseline_fcas      = fcas_revenue_per_mw_year × power_mw
baseline_capacity  = capacity_payment_per_mw_year × power_mw
baseline_total     = baseline_arbitrage + baseline_fcas + baseline_capacity
```

#### 6.9.3 Step 2: 总投资计算

```
total_capex = capex_per_kwh × capacity_mwh × 1000 + grid_connection_cost

其中: capacity_mwh = power_mw × duration_hours
```

#### 6.9.4 Step 3: 逐年现金流模型

```
Year 0:
  net_cash_flow = −total_capex
  cumulative    = −total_capex

Year i (i = 1 to project_life_years):
  deg_factor    = (1 − degradation_rate)^(i−1)

  rev_arbitrage = baseline_arbitrage × deg_factor      // 套利收入随电池衰退
  rev_fcas      = baseline_fcas × deg_factor           // FCAS 同理
  rev_capacity  = baseline_capacity                    // 容量补贴不衰退
  total_rev     = rev_arbitrage + rev_fcas + rev_capacity

  annual_fixed_om = fixed_om_per_mw_year × power_mw + land_lease_per_year
  annual_var_om   = variable_om_per_mwh × capacity_mwh × 365 × 0.85 × deg_factor
  total_opex      = annual_fixed_om + annual_var_om

  net_cash_flow   = total_rev − total_opex
  cumulative     += net_cash_flow
```

#### 6.9.5 Step 4: NPV (净现值)

```
       n
NPV = Σ  (CF_i ÷ (1 + discount_rate)^i )
      i=0

其中 CF_0 = −total_capex, CF_i = net_cash_flow (i年)
```

#### 6.9.6 Step 5: IRR (内部收益率) — 二分法求解

```python
def compute_irr(cash_flows):
    # 找到使 NPV = 0 的折现率 r
    low, high = -0.5, 5.0

    for _ in range(1000):
        mid = (low + high) / 2
        npv = Σ(CF_i / (1 + mid)^i)

        if |npv| < 1e-6:
            return mid         # 收敛
        if npv > 0:
            low = mid          # 折现率太低
        else:
            high = mid         # 折现率太高

    return (low + high) / 2
```

#### 6.9.7 Step 6: 回收期 (Payback Period)

```
payback_year = 最早满足 cumulative_cash_flow ≥ 0 的年份 i
```

#### 6.9.8 Step 7: ROI

```
ROI = Σ(所有年份 net_cash_flow) ÷ total_capex × 100%
```
